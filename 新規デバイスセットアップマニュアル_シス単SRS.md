## セットアップ手順

### ステップ1：アプリにアクセス

#### **PC / スマートフォン（ブラウザ）の場合**

1. ブラウザを開く
2. アドレスバーに以下を入力：
   ```
   https://rokusuke22-hub.github.io/systan-srs/
   ```
3. **Enter キーを押す**

4. ページが読み込まれて、以下のような画面が表示される：
   ```
   シス単 SRS
   2026-03-21
   クラウド接続済（GAS URL が設定されている場合）
   または
   グラウド接続なし（GAS URL が未設定の場合）
   ```

#### **PWA（ホーム画面アイコン）で開く場合**

1. ブラウザでアプリを開いたまま
2. ブラウザメニュー（⋮）をタップ
3. 「アプリをインストール」を選択
4. 「インストール」をタップ
5. ホーム画面に「シス単 SRS」アイコンが追加される

**以降：** ホーム画面のアイコンをタップでアプリ起動

---

### ステップ2：設定画面を開く

1. **ホーム画面が表示されている状態**から始める

2. 画面下部の **「設定」ボタン** をタップ

3. 「設定」画面が表示される：
   ```
   ├─ 1日の復習上限          [50]
   ├─ 卒業判定（間隔が何日以上で卒業）  [30]
   ├─ クラウド同期
   │  ├─ GAS URL           [入力フィールド]
   │  ├─ 今すぐ同期          [ボタン]
   │  ├─ クラウドから取り込み  [ボタン]
   │  └─ 接続テスト（診断）    [ボタン]
   └─ [← ← ←]（戻るボタン）
   ```

---

### ステップ3：GAS URL を設定

#### **GAS URL とは何か**

Google Apps Script（GAS）のデプロイメント URL。
**複数デバイス間でデータを同期するための「サーバーアドレス」** です。

#### **既存デバイスから GAS URL を取得**

1. 既存デバイス（PC など）の設定画面を開く
2. 「GAS URL」フィールドに表示されている URL をコピー

**例：**
```
https://script.google.com/macros/s/AKfycbUTVGH5q0DAwGYeRvaJELT.../useless
```

#### **新規デバイスに GAS URL を入力**

1. 新規デバイスの設定画面を開く

2. 「GAS URL」フィールドをタップ

3. 既存デバイスからコピーした URL をペースト

4. **自動で保存される**

5. 以下のメッセージが表示される：
   ```
   ✅ クラウド接続済
   ```

---

### ステップ4：接続テスト（診断）を実行

#### **目的**

GAS URL が正しく機能しているか確認。

#### **実行方法**

1. 設定画面で **「接続テスト（診断）」ボタン** をタップ

2. ブラウザの開発者ツール（DevTools）を開く
   - **Windows / Linux**：F12
   - **Mac**：Cmd + Option + I

3. **Console タブ** をクリック

4. アラート（ポップアップ）が複数回表示される

#### **正常な結果**

```
✅ POST成功：スプレッドシートにデータ書き込み確認
または
✅ GETフォールバック成功
```

いずれかが表示されていれば OK。

#### **異常な結果**

```
❌ POST失敗：データが書き込まれていません
❌ GETフォールバックも失敗
```

この場合は → [トラブルシューティング](#トラブルシューティング) を参照

## 同期テスト

### 目的

複数デバイス間でデータが正常に同期されることを確認。

### 前提

- **2台以上のデバイス** がセットアップ済み
- 両方で同じ GAS URL が設定されている
- 両方が同じ Google ドライブ（親のドライブ）を共有している

### テスト手順

#### **テスト1：片方向同期（端末A → 端末B）**

**1. 端末A で語を登録**

1. ホーム画面に戻る（「← ← ←」をタップ）
2. **「語を登録する」** をタップ
3. 語番号、フレーズ、日本語訳を入力
   - 例：
     - 語番号：#100
     - フレーズ：example
     - 日本語訳：例

4. **「登録」** をタップ
5. 成功メッセージが表示される

**2. 端末A で同期確認**

1. DevTools（F12）を開いて **Console タブ** をクリック
2. 以下のログが表示されているか確認：
   ```
   [pushCloud START]
   [pushCloud POST]
   [pushCloud SUCCESS]
   ```

**3. 端末B で 30～40秒待機**

待機中の間に、ポーリングが実行される。

**4. 端末B でダイアログが出ているか確認**

```
ダイアログが表示される：
「別の端末で更新があります。
クラウドのデータを取り込みますか？」
```

- ✅ **ダイアログが出た** → 同期が成功している
- ❌ **ダイアログが出ない** → 同期に問題あり

**5. 端末B で「はい」をクリック**

データが端末B に取り込まれる。

**6. 端末A で 30秒以上放置**

端末A のポーリングでは、以下のログが表示される：
```
[POLL] 実行。localTs=XXXX
[POLL CHECK] cloud.ts=XXXX vs local.ts=XXXX
[POLL] 更新なし
```

- ✅ **「更新なし」と出た** → ピンポン問題なし（正常）
- ❌ **ダイアログが再び出た** → ピンポン問題あり

#### **テスト2：逆方向同期（端末B → 端末A）**

テスト1と同じ手順を、端末B で語を登録し、端末A で確認。

#### **テスト3：削除同期**

1. 端末A でホーム画面 → 「登録語一覧」をタップ
2. テスト1 で登録した語（#100）をタップ
3. **「削除」** をタップ

Console で以下を確認：
```
[pushCloud START]
[pushCloud SUCCESS]
```

4. 端末B で 30秒待機
5. ダイアログが出て、削除が反映されていることを確認

## トラブルシューティング

### 問題1：GAS URL が無効と言われる

**症状：**
```
接続テスト実行時に
❌ エラー: 404 Not Found
```

**原因：**
- GAS URL が間違っている
- GAS がデプロイされていない

**解決方法：**

1. 既存デバイスの設定画面から GAS URL をコピーし直す
2. 新規デバイスに**完全にペースト**（途中で切れていないか確認）
3. 接続テストを再実行

### 問題2：ダイアログが出ない（同期されない）

**症状：**
```
端末A で登録 → 端末B で 30秒待機
→ ダイアログが出ない
```

**原因：**
- GAS URL が異なっている
- クラウドのスプレッドシートが共有されていない
- ブラウザキャッシュが古い

**解決方法：**

1. **両端末の GAS URL を確認**
   - 完全に同じ URL か確認
   - コピペで再設定

2. **スプレッドシート共有を確認**
   - 親のGoogleドライブでスプレッドシート（SRS_Data）を確認
   - 子どものアカウントで共有されているか確認
   - 権限：「編集可能」になっているか確認

3. **ブラウザキャッシュをクリア**
   - Ctrl + Shift + Delete（Windows）
   - Cmd + Shift + Delete（Mac）
   - 「キャッシュ済みの画像とファイル」にチェック
   - 「データを削除」をクリック
   - ページをリロード（F5）

4. **Service Worker を削除**
   - DevTools → Application タブ
   - 「Service Workers」をクリック
   - 「Unregister」をクリック

### 問題3：ピンポン問題（ダイアログが何度も出る）

**症状：**
```
端末 A で登録後、
「別の端末で更新があります」というダイアログが
30秒ごとに繰り返し出現
```

**原因：**
- アプリが古いバージョンでセットアップされている
- タイムスタンプの同期がループしている

**解決方法：**

1. **GitHub Pages の最新バージョンをリロード**
   - Ctrl + Shift + Delete でキャッシュ削除
   - Ctrl + F5 で完全リロード

2. **localhost キャッシュをクリア**
   - DevTools → Application タブ
   - Cache Storage をすべて削除
   - Service Workers をすべて Unregister

3. **ページ全体をリロード**
   ```
   https://rokusuke22-hub.github.io/systan-srs/
   ```

### 問題4：画面に「同期ボタン」が表示されない

**症状：**
```
設定画面に
「今すぐ同期」「接続テスト」などのボタンが見当たらない
```

**原因：**
- GAS URL が設定されていない
- アプリが古いバージョン

**解決方法：**

1. **GAS URL を入力**
   - 設定画面で「GAS URL」フィールドを確認
   - 空白なら、既存デバイスから URL をコピー

2. **アプリをリロード**
   - ブラウザを更新（F5）
   - ボタンが表示される

## よくある質問（FAQ）

### Q1：複数の Googleアカウントで使用できますか？

**A：** はい、できます。ただし以下の条件があります：

- **親のデバイス**：親のアカウント
- **子どものデバイス**：子どものアカウント
- **スプレッドシート**：親のドライブに保存し、子どもに共有

### Q2：オフラインでも使用できますか？

**A：** はい。PWA（Progressive Web App）対応なので：

- **オンライン時**：通常どおり同期
- **オフライン時**：ローカルで学習。復帰後に自動同期

### Q3：データはどこに保存されますか？

**A：** 2箇所に保存されます：

- **ローカル**：ブラウザの localStorage（各デバイス）
- **クラウド**：親の Google Drive 内のスプレッドシート

### Q4：万一 GAS URL を忘れた場合は？

**A：** 既存デバイスで確認できます：

1. 既存デバイスで設定画面を開く
2. 「GAS URL」フィールドから URL をコピー

## サポート情報

### ログの確認方法

**DevTools Console を確認：**

1. F12 キーを押す
2. Console タブをクリック
3. 以下のキーワードで検索：
   - `[pushCloud]` → 同期ログ
   - `[POLL]` → ポーリングログ
   - `[fetchCloud]` → クラウド読み込みログ

### エラーが出た場合

1. エラーメッセージをスクリーンショット
2. Console のログをコピー
3. 既存デバイスとの GAS URL を照合
4. GAS デプロイメント URL が最新か確認
